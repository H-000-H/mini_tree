mini_tree 架构总览

三层降维解耦拓扑 · 两段式安全点火 · 7 层纵深防御 · 跨平台验证矩阵

1. 三层降维解耦拓扑

mini_tree 在编译链接层切断"芯片 SDK / RTOS"与"应用业务"之间的符号依赖，实现核心逻辑跨平台无感平替。

宿主应用层 main.cpp / 业务 Service / 外设控制 / 算法控制流 仅依赖 mini_tree 接口库
核心中间件层 core/ · board/ · system_(c|cpp)/ — EventBus, BufferPool, VFS 设备树, Probe, 看门狗
OSAL 操作系统抽象层 osal_freertos.c · osal_rtthread.c · osal_null.c — 统一 Task/Queue/Mutex/Spinlock/Time
微内核与硬件生态层 FreeRTOS 内核 · RT-Thread 内核 · 裸机 SysTick — Kconfig 编译期选择

3. 启动时序 (两段式点火)

Phase 1 — System_Pre_OS_Init()
OS 启动前执行：Meyers Singleton 预触 · BufferPool 初始化 · RTC WDT + Task WDT · EventBus · Bootloop 基线 · DTS Probe · Scrubber CRC 基线
Phase 2 — System_Start_Tasks()
OS 启动后执行：EventBus 分发任务 · Task WDT 监控 · 栈水位监控 · Scrubber 巡检任务
main()
  ├─ platform_init()
  ├─ platform_register_drivers()
  ├─ System_Pre_OS_Init()      // Phase 1
  ├─ System_Start_Tasks()     // Phase 2
  ├─ OSAL_NULL? → mini_tree_system_loop()
  └─ 否则 → vTaskStartScheduler()
阶段时机可访问资源失败后果
Phase 1OS 启动前CPU + 栈 + BSS/DataBootloop 防护介入
Phase 2OS 启动后多任务 + IPC + Queue单任务失败不影响已有服务

4. 核心数据流

EventBus 发布订阅

发布者                    EventBus                    订阅者
   │                         │                          │
   ├─ osal_queue_send(evt) ──┤                          │
   │                         ├─ 快照订阅者列表 (锁保护)
   │                         ├─ ISR → 直接分发
   │                         │   否则 → Queue 投递
   │                         ├─ osal_queue_send(evt) ───┤
   │                         │                          ├─ osal_queue_receive(evt)

BufferPool 无锁分配

分配: __builtin_ctz(bitmap) → CAS(bitmap) → 返回块索引
释放: atomic_fetch_or(bitmap, 1u << index)
特性: O(1), ISR 安全, 零碎片

设备树 Probe 流程

board.dts → dtc-lite.py (Kahn 排序) → .rodata 结构数组
         → DRIVER_REGISTER (构造函数链接) → board_probe_all()
         → 级联初始化 → 失败联动回滚

6. 安全架构 (7 层防御)

L1Bootloop 防护连续崩溃 ≥ 5 次 → 永久锁 Flash
L2RTC 硬件看门狗CPU/总线卡死 → 32kHz 独立时钟物理复位
L3Task WDT3 秒未喂狗 → Core Dump + 硬件复位
L4栈水位监控剩余 < 512 字节 → 两级预警 → 中断闭锁
L5Flash ScrubberCRC 失配 → Bit-Rot 检测 → 安全状态
L6Safe StateOSAL_PANIC / init 失败 → 关中断 + 锁调度器
L7反汇编审查CONFIG_BUILD_DISASM → 指令级验证原子操作

安全状态机

正常运行 ◄──── 喂狗 / WDT 刷新
    │
    ├─ OSAL_PANIC() ────────► Safe State (关中断, 死循环)
    ├─ Task WDT 超时 ──────► Core Dump → 硬件复位
    ├─ RTC WDT 超时 ───────► 物理电源复位
    ├─ Bootloop ×5 ────────► 永久锁 Flash → Safe State
    ├─ Scrubber CRC 失配 ───► Safe State (标记损坏页)
    └─ 栈超限 ─────────────► 中断闭锁 → 硬件复位
